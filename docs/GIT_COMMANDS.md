# 유용한 Git 명령어 모음

이 문서는 프로젝트 구조 변경과 같은 대규모 작업 시 유용한 Git 명령어와 병합 전략을 정리한 것입니다.

## 1. 병합 충돌 해결 전략 (Merge Conflict Strategy)

다른 브랜치와 병합할 때 충돌이 발생할 경우, **현재 브랜치(변경한 구조)**를 우선하거나 **상대 브랜치**를 우선하는 방법입니다.

### 현재 브랜치 내용을 우선하여 병합 (Ours)
구조 변경 작업을 수행한 현재 브랜치의 내용을 유지하고 싶을 때 사용합니다.

```bash
# 병합 시도 시 충돌이 발생하면, 현재 브랜치(HEAD)의 내용을 채택
git merge -X ours <대상_브랜치명>
```

### 상대 브랜치 내용을 우선하여 병합 (Theirs)
상대 브랜치의 변경 사항을 우선하고 싶을 때 사용합니다.

```bash
# 병합 시도 시 충돌이 발생하면, 들어오는 브랜치(대상_브랜치)의 내용을 채택
git merge -X theirs <대상_브랜치명>
```

### 이미 충돌이 발생한 상태에서 특정 파일만 선택하기
병합 도중(`MERGING` 상태) 특정 파일에 대해 우리 쪽 버전을 선택하고 싶을 때:

```bash
# 현재 브랜치 버전으로 덮어쓰기
git checkout --ours <파일경로>
git add <파일경로>

# 상대 브랜치 버전으로 덮어쓰기
git checkout --theirs <파일경로>
git add <파일경로>
```

## 2. 자주 사용하는 Git 명령어

### 상태 확인 및 로그
```bash
# 현재 변경 상태 확인 (가장 많이 사용)
git status

# 커밋 로그 확인 (그래프로 보기)
git log --graph --oneline --all --decorate

# 변경 내용 차이 확인
git diff
```

### 브랜치 작업
```bash
# 브랜치 목록 확인
git branch -a

# 새 브랜치 생성 및 이동
git switch -c <새_브랜치명>
# 또는 (구버전)
git checkout -b <새_브랜치명>

# 브랜치 이동
git switch <브랜치명>
```

### 임시 저장 (Stash)
작업 중인 내용을 커밋하지 않고 잠시 다른 브랜치로 이동해야 할 때 유용합니다.

```bash
# 현재 작업 내용 임시 저장
git stash

# 임시 저장 목록 확인
git stash list

# 가장 최근에 저장한 내용 불러오기 (적용 후 목록에서 삭제)
git stash pop

# 가장 최근에 저장한 내용 불러오기 (목록 유지)
git stash apply
```

### 원격 저장소 동기화
```bash
# 원격 저장소의 변경 사항 가져오기 (병합 안 함)
git fetch --all

# 원격 저장소의 변경 사항 가져와서 현재 브랜치에 병합
git pull origin <브랜치명>

# 로컬 변경 사항 원격 저장소에 올리기
git push origin <브랜치명>
```

### 작업 되돌리기 (주의 필요)
```bash
# 스테이징된 파일(add 한 파일)을 다시 내리기
git restore --staged <파일경로>

# 작업 중인 파일의 변경 사항을 모두 취소하고 마지막 커밋 상태로 되돌리기
git restore <파일경로>
